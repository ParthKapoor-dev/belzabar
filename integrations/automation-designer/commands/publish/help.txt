belz ad publish <uuid> --note "<what changed>" [flags]

Publish a draft — promotes the current draft contents to the published
version of the same method. Returns the published UUID and a version-sync
summary.

A change note is MANDATORY: belz records it against the chain's changelog
before the publish call, so every published change is logged.

Flags:
  --note "<text>"  Required. The change note recorded against the chain.
  --yes            Skip the interactive confirmation prompt
  --llm            Emit a JSON envelope; --yes is required in this mode

Safety:
  - A missing --note aborts before any server call (MISSING_CHANGE_NOTE).
  - The change note is POSTed to the changelog BEFORE the publish call.
  - Always routes through resolveDraftTarget: if you pass a PUBLISHED UUID
    belz automatically switches to the linked draft before publishing.
  - Re-fetches both draft and published after the publish call and reports
    whether they are in sync.

See also: belz ad changelog — record/list change notes without publishing.
