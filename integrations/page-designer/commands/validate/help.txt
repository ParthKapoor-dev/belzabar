Usage: belz pd validate <INPUT> [flags]

Validate a Page Designer page or component against common error patterns.
Checks for orphan bindings, unused variables, invalid components, and more.

Arguments:
  <INPUT>             One of:
                      • App page URL:  https://<domain>/pages/<path>
                      • PD page URL:   https://.../ui-designer/page/<id>
                      • PD symbol URL: https://.../ui-designer/symbol/<name>
                      • Bare hex ID:   48b75e11e166e6b19e5796d59a936228
                      • Component name: n_s_staff_email_correspondence

Validation Checks:
  ORPHAN_BINDING          {%var%} used but variable not defined (error)
  UNUSED_VARIABLE         Variable defined but never referenced (warn)
  FORM_FIELD_PROPS        exp-form-field with props instead of field (error)
  CHILDREN_NOT_ARRAY      children is object instead of array (error)
  TABLE_NO_DATASOURCE     exp-data-table missing datasourceState (warn)
  INVALID_SLIDE_TOGGLE    mat-slide-toggle used (no runtime declaration) (error)
  INVALID_EXPANSION_HEADER  mat-expansion-panel-header used (error)
  MISSING_ONINIT_VAR      HTTP trigger onInit but no onInit variable (warn)
  EMPTY_EVENT_META        eventMeta: {} on variable-triggered call (warn)
  DYNAMIC_COLS_INITIAL    exp-data-table with [columns] + initialValue (warn)

Global Flags:
  --env <name>        Set active environment. (default: nsm-dev)
                      Available: nsm-dev | nsm-qa | nsm-uat
  --llm               Output structured JSON envelope for scripting/LLM use.
  --help, -h          Show this help message.

Examples:
  belz pd validate 48b75e11e166e6b19e5796d59a936228
  belz pd validate n_s_staff_email_correspondence
  belz pd validate https://nsm-dev.nc.verifi.dev/ui-designer/page/48b75e11e166e6b19e5796d59a936228
  belz pd validate 48b75e11e166e6b19e5796d59a936228 --llm
