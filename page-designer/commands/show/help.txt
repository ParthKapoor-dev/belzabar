Usage: belz pd show <INPUT> [flags]

Display the structure and metadata of a Page Designer page or component.
The config is fetched and cached automatically (5-minute TTL).

Arguments:
  <INPUT>             One of:
                      • App page URL:  https://<domain>/pages/<path>
                      • PD page URL:   https://.../ui-designer/page/<id>
                      • PD symbol URL: https://.../ui-designer/symbol/<name>
                      • Bare hex ID:   48b75e11e166e6b19e5796d59a936228
                        (auto-detects page vs. component)
                      • Component name: n_s_staff_email_correspondence

Flags:
  --vars              Show all variables (user-defined + derived).
  --http              Show all HTTP service calls.
  --components        Show the legacy (flat-name) layout tree.
  --tree              Show the full layout tree with node-kind badges
                      ([FORM] [TABLE] [BTN] [SYM] [LAYOUT] [-]). Highlights
                      known pitfalls inline (USES-PROPS!, DYN-CLASS!, NO-DS).
  --node <id>         Dump kind-specific detail for a single node by id
                      (field vs props, bindings, validations, event wires).
  --var-graph         Show the per-variable write/read/derive/trigger map.
                      Flags "dead" variables (no writers AND no readers).
  --var-detail <name> Show full config for a specific variable or derived.
  --http-detail <N>   Show full config for the Nth HTTP call (1-indexed).
  --full              Include all sections + all details.
  --recursive, -r     Include recursive dependency tree and all AD method IDs.
                      Requires components.json in the current directory.
  --force             Bypass cache and fetch fresh config.
  --raw               Include raw configuration string and source API fields.
  --open, -o          Open the editable draft URL in your default browser
                      (uses the draft id for pages, the component name for
                      symbols).
  --help, -h          Show this help message.

Global Flags:
  --env <name>        Set active environment. (default: nsm-dev)
                      Available: nsm-dev | nsm-qa | nsm-uat
  --llm               Output structured JSON envelope for scripting/LLM use.

Examples:
  belz pd show 48b75e11e166e6b19e5796d59a936228
  belz pd show https://nsm-dev.nc.verifi.dev/ui-designer/page/48b75e11e166e6b19e5796d59a936228
  belz pd show n_s_staff_email_correspondence
  belz pd show 48b75e11e166e6b19e5796d59a936228 --vars
  belz pd show 48b75e11e166e6b19e5796d59a936228 --http
  belz pd show 48b75e11e166e6b19e5796d59a936228 --components
  belz pd show 48b75e11e166e6b19e5796d59a936228 --var-detail showLoader
  belz pd show 48b75e11e166e6b19e5796d59a936228 --http-detail 1
  belz pd show 48b75e11e166e6b19e5796d59a936228 --full
  belz pd show 48b75e11e166e6b19e5796d59a936228 --force --llm
  belz pd show 48b75e11e166e6b19e5796d59a936228 --tree
  belz pd show 48b75e11e166e6b19e5796d59a936228 --node __node_id_submit_btn
  belz pd show 48b75e11e166e6b19e5796d59a936228 --var-graph
  belz pd show 48b75e11e166e6b19e5796d59a936228 --open
  belz pd show AHS_DMV_Profile_Detail -o
