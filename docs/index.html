Unified CLI

Native binary built with Bun. Inspect methods, run tests with per-step trace logs, execute SQL queries, search pages, and trigger cross-environment migrations. Namespaced under belz ad, belz pd, belz tw, and belz migrate. Every command supports --llm for structured JSON output that AI agents consume directly.

Smart Extension

Manifest V3 extension for Chrome & Edge. JSON bulk editors, CodeMirror textarea modals, one-click output copy, title updater, rich link copy (Shift+L), and Ctrl+Shift+Enter to run tests.

Zero Auth Latency

Session files persist Bearer tokens per environment. On 401, the auth module re-logs in automatically and retries the failed request — no manual token management ever.

AI Ready (ACP Bridge)

A Next.js web app bridges the Agent Client Protocol over stdio to coding agents (Claude Code, Gemini, Codex, OpenCode). Supports streaming text, tool calls, permission gates, managed terminals, and constrained filesystem access — all in a single singleton Node.js process.

Web Dashboard

Next.js 16 app with unified Ctrl+K search across AD & PD, VIN lookup, and curl → AD autofill tool.

Teamwork Integration

Fetch tasks, comments, assignees, tags, and workflow stages from Teamwork project management.

SQL Engine

SELECT, INSERT, UPDATE, DDL against NSM databases. Interactive TUI REPL with history, pagination, and modes.

Request Lifecycle

01

Entry — cli.ts or cli-build.ts receives process.argv

02

Global Flags — --llm stripped → sets outputMode. --env <name> stripped → sets active environment

03

Routing — runNamespacedCli() splits on first arg (ad / pd / tw / migrate / config / web) and looks up the command in the registry map

04

Parse — CommandModule.parseArgs() validates CLI args into typed TArgs

05

Auth — apiFetch() loads Bearer token from ~/.belz/sessions/<env>.json; if absent, calls login() first

06

API — HTTP request to active NSM environment URL. On 401, session cleared and retried once

07

Execute — Raw response parsed into domain types by module-level parsers

08

Render — renderHuman() outputs colored tables; renderLLM() outputs full CommandEnvelope as JSON

Auth & HTTP

Sessions at ~/.belz/sessions/<env>.json. login() POSTs to /do/login, extracts Bearer token. apiFetch(path, options) is the single HTTP entry point — attaches auth headers, supports Bearer/Raw/None modes, auto re-authenticates + retries on 401.

// Auth flow inside apiFetch:
                            1. loadSession() → attach Authorization header
                            2. fetch(url, { headers })
                            3. if (response.status === 401):
                            clearSession(); await login(); retry once

CommandModule Interface

interface CommandModule<TArgs,
                            TData> {
                            schema?: string; // e.g. "ad.show"
                            version?: string; // default "2.0"
                            parseArgs?(rawArgs, ctx): TArgs;
                            execute(args, ctx): Promise<CommandResult<TData>>;
                            presentHuman?(envelope, ui): void;
                            }

                            interface CommandEnvelope<TData> {
                            schema: string; version: string; ok:
                            boolean;
                            command: string;
                            data: TData | null;
                            error: { code: string; message: string;
                            details?: unknown } | null;
                            meta: { env: string; durationMs: number;
                            warnings: string[] };
                            }

                            // Helper factories:
                            ok(data, meta?) → { ok: true, data,
                            meta }
                            fail(code, msg) → { ok: false, error:
                            { code, message } }

Library Modules

parser.ts — parseMethodResponse(raw) unwraps the doubly-serialized jsonDefinition string. Normalises method name (inner takes precedence over aliasName).

hydrator.ts — ServiceHydrator fetches + caches AutomationDefinition objects. Core IDs (hyphenated) cached permanently; others expire in 1 hour. findInputLabel(def, inputId) does recursive search; flattenInputs(def) returns flat list with depth.

method-finder.ts — Builds full index by paging /automation/services (2000/page) + /automation/methods (100/page per category). 7-day cache. Scoring: exact (1000) → starts-with (900) → word-boundary (850) → contains (760) → fuzzy with Levenshtein (600–700). Weights: methodName 1.0, aliasName 0.95, categoryName 0.85.

input-collector.ts — File mode: reads JSON, validates required fields. Interactive: type-aware prompts (BOOLEAN yes/no, JSON parse validation).

payload-builder.ts — Injects testValue + automationApiId into method payload for test execution.

error-parser.ts — Identifies error type: DB (databaseError), CODE (message + stackTrace), UNKNOWN. Returns ParsedError with summary + detail.

Client State

Managed by useReducer hook. State persisted to localStorage, reconciled with server sessions on mount. SSE streaming from /ai/api/sessions/{id}/prompt parsed line-by-line and dispatched as reducer actions for efficient partial re-renders.

CLI Binary

Bun workspace at cli/. Compiles to a single native binary at ~/.local/bin/belz.

Dev vs. Prod Entry Points

bin/cli.ts — Dev

  • Help textresolved from help.txt

bin/cli-build.ts — Prod

  • Commandsstatic registry-*.ts
  • Help textembedded in registry-help.ts
  • Compiled tosingle native binary
  • Runtime FSnone (all bundled)

                            cd cli && bun run generate

                            # Full install (credentials + binary)
                            bash cli/scripts/install.sh

                            # Clean wipe + fresh install
                            bash cli/scripts/reinstall.sh
                            # or: bun run reinstall

Install Script

01

Install npm dependencies (bun install)

02

Run generate to rebuild registries

03

Compile binary via bun build --compile --minify --sourcemap

04

Copy binary to ~/.local/bin/belz

05

First run only: prompt for credentials or read --env-file

06

Migrate old sessions from ~/.belzabar-cli/sessions/ if present

07

Build web app and install to ~/.belz/web/
