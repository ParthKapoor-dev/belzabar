Getting started
Install the CLI
Requires Bun ≥ 1.x. The install script handles dependencies, builds the binary, and walks through credential setup.

Quick install

1

Install Bun (if needed)

curl -fsSL https://bun.sh/install | bash
2

Clone and install

The install script builds the binary and prompts for credentials.

git clone <repo-url> belzabar && cd belzabar bun run install
3

Add to PATH

export PATH="$PATH:$HOME/.local/bin"
4

Verify install

belz --help
5

Load the browser extension (optional)

Open chrome://extensions, enable Developer Mode, click Load unpacked, and select the extension/ directory.

Module Dispatch Type: Regular vs Passthrough

Regular Module  ad pd

Has a commands map. The runner handles routing to sub-commands.

dispatches "get" in ADCommandRegistry →

Passthrough Module  migrate

Has a single command module. Passes full args through — the command handles its own sub-routing.

What is web/?

Purpose

A Next.js 16 web application that provides a browser UI for connecting to local coding agents via the Agent Client Protocol (ACP). Users can open multiple chat sessions against different working directories, each backed by a live agent subprocess.

Key constraint: Browsers cannot spawn processes or use stdio directly. The web app solves this with a server-side bridge that lives in Next.js API routes (Node.js runtime) and proxies browser requests ↔ agent stdio.

ACP — Agent Client Protocol
ACP is a JSON-RPC 2.0 protocol that defines how a client (this web app) communicates with an agent (e.g. opencode). The agent runs as a child process; communication happens over stdin/stdout as newline-delimited JSON (NDJSON), or over HTTP if the agent starts an HTTP server and announces its port.

The ACP Bridge
Why it exists

Browsers cannot use Node.js child_process or pipe stdio. The bridge is a singleton class that lives on the Next.js server, owns all agent subprocesses, and exposes them through HTTP.

Singleton pattern
// lib/acp-bridge.ts — bottom of file const g = global as typeof global & { _acpBridge?: AcpBridge }; export const bridge = g._acpBridge ?? new AcpBridge(); if (process.env.NODE_ENV !== "production") { g._acpBridge = bridge; // survive hot-reload in dev }
Hot-reload safety: In development, Next.js re-evaluates modules on change. Storing the instance on global ensures the same bridge (and all its running agent processes) survive a hot-reload without orphaning subprocesses.
Internal state per session
type SessionEntry = { id: string; // our UUID agentName: string; // e.g. "opencode" agentCommand: string; // e.g. "opencode acp" cwd: string; name: string | undefined; // from session_info_update connection: ClientSideConnection; child: ChildProcess; acpSessionId: string; // agent's session UUID status: "idle"|"running"|"closed"; createdAt: string; subscribers: Set<(e: BridgeEvent) => void>; pendingPermission: PendingPermission | undefined; terminals: Map<string, ManagedTerminal>; };
Definite-assignment trick: let entry!: SessionEntry is used so the ACP connection callbacks can close over entry before it's assigned — safe because callbacks only fire during prompt(), which happens after entry is set.

Client State Management
useReducer architecture (use-sessions.ts)

All session + workspace state lives in a single useReducer. dispatch is stable across renders, making it safe to capture in async SSE loops without stale closure issues.

type SessionsState = { slots: SessionSlot[]; workspaces: Workspace[]; initialized: boolean; // true after server reconcile } // Actions "INIT" // load from localStorage "SET_INITIALIZED" // server reconcile done "ADD_SLOT" // new session created "UPDATE_SLOT" // patch any field by id "UPDATE_LAST_ASSISTANT" // stream chunk into message "APPEND_MESSAGES" // push user+assistant msgs "REMOVE_SLOT" // close session "ADD_WORKSPACE" "UPDATE_WORKSPACE" // collapse/rename "REMOVE_WORKSPACE" // orphans → default
SessionSlot shape
type SessionSlot = { // — from server SessionInfo — id: string; agentName: string; agentCommand: string; cwd: string; name?: string; // from ACP session_info_update status: "idle"|"running"|"closed"; createdAt: string; // ISO 8601 workspaceId: string; // — client-only — connectionStatus: "live" | "disconnected"; messages: Message[]; isRunning: boolean; pendingPermission: PendingPermission | null; connectError: string | null; connecting: boolean; };
connectionStatus: "live" = server session exists (can send prompts). "disconnected" = only in localStorage (server restarted) — UI shows history but disables input.
Initialization sequence (on page load)
  1. Browser renders AiShell. useSessions() + useSettings() mount. Initial state: empty slots, default settings.
  2. useEffect fires: Read ai:sessions + ai:workspaces from localStorage → dispatch INIT. All slots start as connectionStatus: "disconnected".
  3. Simultaneously: GET /ai/api/sessions hits the bridge. For each server session, dispatch UPDATE_SLOT({ connectionStatus: "live" }). Sessions only on server (other tab) get ADD_SLOT.
  4. After fetch resolves: dispatch SET_INITIALIZED → state.initialized = true. Loading screen disappears.
  5. useSettings loads: Reads ai:settings from localStorage → settingsLoaded = true.
  6. Auto-connect: If both initialized + settingsLoaded, and settings.autoConnect = true, and no live sessions exist → call createSession(defaultCwd, mainAgent) → navigate to /ai/{id}.

SSE Streaming Implementation
Server side (prompt/route.ts)
// TransformStream bridges the async prompt() call // to a streaming HTTP response const { readable, writable } = new TransformStream<Uint8Array>(); const writer = writable.getWriter(); const encoder = new TextEncoder(); const emit = (event: BridgeEvent): void => { writer.write( encoder.encode(`data: ${JSON.stringify(event)}\n\n`) ).catch(() => {}); }; // Run prompt async, close stream when done bridge.sendPrompt(id, message, emit) .finally(() => writer.close().catch(() => {})); return new Response(readable, { headers: { "Content-Type": "text/event-stream", ... } });
Client side (use-sessions.ts)
// Fetch returns a streaming ReadableStream const reader = res.body.getReader(); const decoder = new TextDecoder(); let buf = ""; while (true) { const { done, value } = await reader.read(); if (done) break; buf += decoder.decode(value, { stream: true }); const lines = buf.split("\n"); buf = lines.pop() ?? ""; for (const line of lines) { if (!line.startsWith("data: ")) continue; const ev = JSON.parse(line.slice(6)) as BridgeEvent; // dispatch uses stable ref — safe in async loop dispatch({ type: "UPDATE_LAST_ASSISTANT", id, fn }); } }
Stale closure safety: dispatch from useReducer is guaranteed stable by React. The SSE loop closes over dispatch and the session's id (captured at sendPrompt call time) — never over mutable state.
Workspaces & Sessions Hierarchy

Workspace

type Workspace = { id: string; name: string; createdAt: string; collapsed: boolean; }

Always has a "default" workspace named "main". Cannot be deleted. Orphaned sessions on workspace delete fall back to it.

Sidebar structure

sessions [🔍] [⊞ new workspace] ────────────────────── ▼ main [+] ── today ────────────── ● belz-ai 2:30pm ── yesterday ────────── ○ sandbox Mar 6 ────────────────────── ▶ work (collapsed) ────────────────────── ⚙ settings (⌘,)

Date grouping

Within each workspace, sessions are sorted by createdAt descending and grouped into date buckets:

  • "today" — same calendar day
  • "yesterday" — prior calendar day
  • "Mon Mar 3" — formatted for older dates

Separators only shown when sessions span multiple days.

How To…

Add a new agent

  1. Add entry to AGENT_REGISTRY in lib/acp-types.ts
  2. It automatically appears in Settings → Agent Profiles dropdown

Add a new BridgeEvent type

  1. Add variant to BridgeEvent union in lib/acp-types.ts
  2. Emit it in the relevant ACP callback in lib/acp-bridge.ts
  3. Handle it in the SSE loop in hooks/use-sessions.ts → dispatch reducer action

Add a new setting

  1. Add field to AppSettings type + DEFAULT_SETTINGS in hooks/use-settings.ts
  2. Add UI for it in app/ai/settings/page.tsx
  3. Read from useSessionsContext().settings wherever needed
